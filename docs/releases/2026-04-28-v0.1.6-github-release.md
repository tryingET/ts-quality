# GitHub release draft — ts-quality v0.1.6

## Title

`ts-quality v0.1.6 — production-clean package presentation and first bounded-review CLI guidance`

## Release body

`ts-quality` v0.1.6 turns the successful Trusted Publishing path into a cleaner first-user package experience.

### Added

- Added first-user CLI help guidance so `ts-quality --help` now shows the first bounded review path: initialize, configure evidence, run `check --changed ... --run-id ...`, then inspect `explain` and `report` against the exact run.
- Expanded command-specific help for the core review, projection, witness, attestation, authorization, amendment, and materialization commands with preconditions, artifact locations, and projection/authority boundaries.

### Changed

- Updated the product posture snapshot to reflect the production-clean packaged README, first-user CLI help, and the remaining production-readiness gap around first-witness adoption.

### Fixed

- Fixed the packaged npm README presentation so the staged package strips repo frontmatter, rejects stale versioned README headings, and starts with a published-package-first quickstart.
- Fixed public release verification so freshly published versions can be checked immediately without changing the maintainer's global npm release-age policy.

### Verification before release

```bash
npm run verify --silent
RELEASE_TAG=v0.1.6 GITHUB_REF_TYPE=tag npm run release:intent:check --silent
```

npx -p ts-quality@0.1.6 ts-quality --help
