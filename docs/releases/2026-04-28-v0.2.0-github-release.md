# GitHub release draft — ts-quality v0.2.0

## Title

`ts-quality v0.2.0 — deterministic trust for TypeScript changes`

## Release body

This release is prepared through the repo's GitHub Release authority path. Publishing the GitHub Release for tag `v0.2.0` triggers `.github/workflows/publish.yml` in the `npm-publish` environment, which validates the tag/version contract, re-runs deterministic package proof, publishes the staged package to npm through Trusted Publishing/OIDC, verifies public installability, and attaches the proven tarball to the GitHub Release.

### Highlights

- Bumps the run artifact schema to `0.2.0` for additive adoption-readiness surfaces.
- Makes surviving mutants actionable with exact site metadata, original/replacement snippets, focused command context, and next-assertion hints.
- Creates the LCOV parent directory before configured coverage generation and writes coverage-generation sidecars.
- Warns when LCOV covers built output (`dist/**`, `lib/**`, `build/**`, or runtime mirrors) but changed TypeScript source under `src/**` is missing source-mapped coverage.
- Adds confidence breakdowns and next-evidence-action artifacts so low merge confidence has an inspectable arithmetic trail and an immediate remediation target.
- Adds read-only `ts-quality doctor` diagnostics plus `init` presets for Node test, TypeScript/dist, and Vitest-style adoption.
