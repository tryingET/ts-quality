# GitHub release draft — ts-quality v0.1.5

## Title

`ts-quality v0.1.5 — project-standard Trusted Publishing tuple`

## Release body

This release aligns `ts-quality` with the same npm Trusted Publishing shape used by our other projects: GitHub Release remains the release intent, while npm recognizes the publisher through workflow filename `publish.yml` and the GitHub Actions environment `npm-publish`.

### Changed

- Renamed the release publication workflow from `.github/workflows/release.yml` to `.github/workflows/publish.yml`, matching the npm Trusted Publisher workflow filename `publish.yml`.
- Added the GitHub Actions environment `npm-publish` to the npm publication job, matching the npm Trusted Publisher environment field.
- Updated release workflow docs, npm publishing checklist, release checklist, generated release-note scaffolding, and regression tests so future releases do not drift back to the old `release.yml` / blank-environment tuple.
