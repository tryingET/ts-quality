# Project Model

This project captures:
- Purpose / mission / vision
- Product posture when the vision-to-strategy bridge needs a high-level maturity snapshot
- Strategic + tactical goals + operating plan
- Constraints + resources
- System4D (Container/Compass/Engine/Fog)

## Direction document roles

- `purpose.md` / `mission.md` / `vision.md` define durable product intent.
- `product_posture.md` is a high-quality status-bearing bridge for product maturity and convergence; it is not a task log and should not become mandatory for every repo.
- `strategic_goals.md`, `tactical_goals.md`, and `operating_plan.md` decompose posture and direction into bounded delivery waves.
- Repo-local AK tasks remain live execution authority.
